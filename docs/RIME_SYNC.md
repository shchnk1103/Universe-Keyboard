# RIME Portable Sync

> **Status:** Product Contract accepted; RIME 标准同步主路径实现中，验收证据待补齐
>
> **Assignment:** [`RIME-SYNC-001`](assignments/rime-sync-001.md)
>
> **Architecture decision:** [ADR 0013](architecture/decisions/0013-rime-standard-sync-interoperability.md), ADR 0012 and [ADR 0019](architecture/decisions/0019-app-notification-and-toast-settings.md)

## Product Goal

让用户在 iPhone、macOS、Windows、Linux 和 Android 上复用同一套可移植 RIME 设置，同时保持离线输入、用户数据安全和主 App / Keyboard Extension 现有边界。

“云同步”是主 App 的数据操作能力，不是键盘按键路径能力。Keyboard Extension 不访问网络、不扫描同步目录，也不负责冲突处理或部署。

## Product Decision

采用“RIME 标准同步 + Universe 私密设置包”的双层方案：

1. **RIME 标准同步** 是跨 App 的主路径。用户选定一个本地/文件提供器目录；Universe 将它写为 librime 的 `sync_dir`，首次由用户在主 App 手动发起官方 `sync_user_data`，之后可按用户的自动同步设置继续维护。
2. **Universe 私密设置包** 是辅助路径。它继续以端到端加密同步本 App 的字段级设置；本地文件夹和 WebDAV 都可承载它，但其他 RIME 前端不会读取它。
3. **WebDAV** 仅承载 Universe 私密设置，除非用户在其他平台将它挂载为真实本地目录；不能被表述为其他 RIME 前端可直接使用的 `sync_dir`。
4. **CloudKit / iCloud** 是后续 Apple 生态适配器。只有 Apple Developer Program、iCloud container、entitlement 和真实设备环境可用后才能进入实现和验证。

当用户用系统文件选择器指定标准同步目录时，主 App 必须在选择回调内立即取得目录的安全作用域，并一直持有到读写预检和 bookmark 保存完成；之后才允许异步任务使用持久化 bookmark 重新获取访问权。这样 iCloud Drive 和第三方文件提供器目录不会因回调结束而失效。

## RIME Standard Sync Main Path

标准层遵循 librime 的既有语义：用户词典以 `*.userdb.txt` 快照合并，用户目录中非自动生成的 YAML/TXT 按设备目录单向备份。它不复制或覆盖运行中的 `*.userdb*`，也不会自动将另一台设备的 YAML 覆盖到本机。

首次标准同步只允许用户在主 App 明确确认后手动发起。成功的初始同步只解锁自动同步能力，不替用户开启任何自动开关；用户必须在页面中自行打开“自动同步”。用户首次主动开启总开关时，“RIME 标准同步”和“Universe 设置同步”两个子项默认开启，之后可独立关闭任一子项，并选择每天或每 7 天一次。关闭最后一个子项会同时关闭总开关，避免界面显示已自动同步却没有任何实际内容。标准同步只在 iOS 分配后台处理时间、键盘活动心跳已失效时运行；Universe 设置同步可在主 App 前台机会运行，也可接在一次标准后台同步之后。两者共享用户选择的冷却时间，不进入 Keyboard Extension 或按键路径，也不承诺固定时刻或实时完成。标准资料不由 Universe 端到端加密，其存储安全由用户选择的文件提供器或已挂载加密卷决定。

## Universe Private Settings Scope

V1 只同步可恢复、可解释的管理型设置，不同步配置文件或用户输入历史：

- Universe Keyboard 管理的 RIME 设置，例如当前方案、候选数量、简繁偏好、模糊音、高级输入和候选学习开关；
- 字段级逻辑版本、设备 ID、格式版本和条件写入所需元数据。

V1 不同步：

- `*.userdb*` 运行数据库；
- `*.userdb.txt` 用户词典快照（它属于 RIME 标准层）；
- 已编译的 `build/`、下载缓存、日志、诊断、Typing Intelligence、Typo 学习数据；
- 平台专属 UI 配置，除非文件被明确标记为对应平台；
- RIME 二进制、Lua/OpenCC 运行资产或整个方案安装包。
- 自定义 YAML / TXT 配置文件；它们需要独立的路径、大小、staging 和恢复安全门禁。

候选学习的跨设备合并属于 RIME 标准层：它通过 librime 的用户词典文本快照进行，不复制运行中的数据库。候选学习数据不进入 Universe 私密设置包；若未来扩大私密包范围，仍需单独同意、隐私评审和恢复/跨进程协调验证。

## Portable Package Contract

V1 同步根目录使用稳定、大小写敏感、跨平台安全的相对路径：

```text
universe-rime-sync/
  format.json
  profiles/default/settings.json
```

规则：

- `format.json` 声明格式版本、加密套件和最小兼容客户端版本；
- 管理型设置按字段记录逻辑版本与设备 ID，离线合并不依赖设备时钟准确性；
- 同字段并发修改按逻辑版本与设备 ID 确定性收敛，不同字段都保留；
- WebDAV 使用 ETag 条件写入并在冲突后重新获取、合并和重试，本地文件夹使用内容摘要实现同等的陈旧写入保护；
- 应用配置由主 App 串行执行，并触发正常 RIME 部署；部署失败时键盘继续使用上一次成功部署的配置，并在主 App 提供重试反馈；
- 未来加入文件同步时必须使用内容哈希、不可变对象、staging、路径/大小校验、冲突副本和恢复快照，不得直接扩展当前设置应用路径。

## Security And Privacy

- Universe 私密设置必须在设备端加密后上传；传输服务只能看到密文、固定最小元数据和对象大小。
- V1 使用独立同步密钥；密钥保存在设备安全存储中，并提供一次性恢复码。服务密码与内容加密密钥不得相同。
- 未开启同步时不发送任何 RIME 数据。
- 开启同步必须清楚列出同步内容、目的地和删除方式。
- 删除远端数据与关闭本地同步是两个独立动作，均需明确反馈。
- 首次标准 RIME 同步只能在用户明确确认后进入选定目录。后续自动同步仅重用同一 librime 快照合并和设备备份路径；任何自动配置导入、覆盖或更宽数据范围均需新的 Product、Architecture 和 Quality Gate。

## Main-App UX

设置首页新增独立的“数据与同步”分组：

- 图标：`icloud` 或语义等价的系统云图标；
- 标题：`RIME 云同步`；
- 副标题：未配置时显示“在多台设备使用同一套设置”，启用后显示稳定状态与最近成功时间。

详情页使用原生 `Form`，保持现有 Settings 风格：

1. **同步状态**：当前状态、最近成功时间、唯一的 `立即同步` 操作。选择 RIME 标准文件夹时，该操作依次执行官方用户资料同步和 Universe 私密设置同步；WebDAV 仅执行后者。
2. **同步方式**：RIME 标准文件夹是主路径；私密 WebDAV 是辅助路径；CloudKit 仅在 capability 实际可用的构建中显示。
3. **同步内容**：标准层列出用户词典快照与 YAML/TXT 备份规则；私密层列出加密字段设置。
4. **安全**：标准层的明文兼容边界、私密层的端到端加密状态及恢复码导出/导入。
5. **管理**：断开连接、删除云端数据。
6. **自动同步**：初次成功后只解锁总开关，由用户自主开启。用户主动开启时两个子开关默认开启；用户可独立关闭任一子项并选择“每天”或“每 7 天”。关闭最后一个子项必须同步关闭总开关。子开关仅控制自动行为；`立即同步` 始终完整同步当前目的地支持的两部分。页面必须用普通用户能理解的文字说明两部分内容、运行机会和安全边界，并明确 iOS 只会在合适的空闲时间运行、键盘使用时会跳过标准同步，不能承诺固定时刻。
7. **通知与提醒**：同步通知覆盖手动和自动操作，属于全 App 的通知类别，不嵌套在自动同步开关下。类别开启后显示“RIME 标准同步”和“Universe 设置同步”两个通知子项；它们只决定哪些同步过程需要提醒，不决定实际执行哪些同步。RIME 页面与“通知与提醒”页面共享同一状态；权限、总开关、合并通知、前台 Toast 优先和迁移遵循 [`APP_NOTIFICATIONS.md`](APP_NOTIFICATIONS.md)。

使用现有 `SettingsNavigationLink`、`AppActionButton` 和全局底部 toast。不要使用大型 hero、渐变、营销插画或常驻操作结果区。

同步状态由一个主 App 拥有的可观察 model 统一表达；页面只保留导航、临时输入和弹窗选择等局部状态。网络、重试、离线合并和恢复策略放在注入的服务中，不从 SwiftUI `body` 发起。加载、同步中、成功、离线、认证失败、密钥错误、冲突和不可用状态都需要确定性的 Preview fixture。

选择新的标准文件夹前，主 App 必须在文件提供器协调范围内完成最小的写入、读回和删除预检，随后才保存 bookmark 与显示名称。预检、bookmark 或后续同步发生目录访问失败时，旧目录信息可以保留作恢复参考，但同步必须暂停；界面必须明确说明新目录未生效或目录已失效，不能继续向旧目录同步。诊断日志只记录阶段和稳定错误码，不记录目录路径、文件名、词典内容、密钥或用户输入。

## Synchronization Semantics

- RIME 标准层的首次同步只允许用户在主 App 中手动确认。完成首次成功同步后只记录自动同步资格和冷却起点，不改变总开关或子开关；用户必须明确打开总开关。用户主动打开总开关时，若两个子项都关闭或尚无存储值，则默认开启两项；若至少一项仍开启，则保留原选择。
- “RIME 标准同步”子开关控制 `BGProcessingTask` 是否在用户选定的冷却周期后调用 `sync_user_data`；开始前必须确认主 App 配置仍有效、共享文件夹可访问、冷却周期已结束且键盘活动心跳已失效。
- “Universe 设置同步”子开关控制主 App 启动、回到前台和设置变更防抖后的私密设置维护，以及一次标准后台同步完成后是否继续维护私密设置。前台路径必须服从同一个每天或每 7 天冷却时间，并按尝试时间节流，避免失败时反复重试。不得承诺实时或固定后台频率。
- 关闭最后一个自动同步子项时，总开关必须立即关闭并取消后台调度；不存在“总开关开启但没有同步内容”的持久状态。
- 手动 `立即同步` 对用户是一个操作，不受两个自动子开关影响：标准文件夹先执行 RIME 标准层，再执行私密设置层；任一步失败即停止并通过全局 Toast 展示明确阶段和错误。一次成功的手动同步会重置前台和后台的共享冷却起点，但不会把用户明确关闭的自动开关重新打开。
- 用户开启 App 通知总开关和 RIME 云同步类别并授予系统权限后，可分别选择 RIME 标准资料和 Universe 设置的通知。两个通知子项同时开启时，一次完整操作合并为一组通知；只开启一项时，只在该部分真实开始、完成、失败或取消时提醒。通知子项不得改变自动或手动同步范围，通知文案不得用局部成功暗示全部同步完成，也不得把未订阅部分的失败表述为已订阅部分失败。断开或重选同步目录不得重置通知偏好。
- 所有同步操作串行化并支持重复调用；中断后从最后一个已验证对象继续。
- 管理型设置只写入现有偏好入口；正常部署成功前不替换键盘当前可用的 RIME 部署。未来配置文件导入必须先建立可验证恢复快照。
- Extension 正在使用 RIME 时，不替换 `Rime/user` 或运行用户数据合并。
- V1 配置同步可以在不接触运行中 `userdb` 的前提下交付。

## Acceptance

- 同一 RIME `sync_dir` 在 iPhone、macOS、Windows、Linux 和 Android 的目标前端上可完成官方词典快照合并，并保留每设备 YAML/TXT 备份。
- Universe 私密设置包在兼容客户端或兼容工具上可往返，不丢失未知字段。
- 离线并发修改产生确定性结果且不静默覆盖文件。
- 无 Apple Developer Program 时，WebDAV 与本地文件夹路径仍可开发和验证。
- CloudKit 构建只有在 entitlement、container、签名和设备证据均成立时才宣称可用。
- 网络失败、认证失败、密钥错误、损坏包、不兼容版本、存储不足和部署失败均保持本地输入可用。
- 同步、哈希、加密、文件扫描和部署不进入键盘按键热路径。
- iOS 的 `earliestBeginDate` 只是最早执行时间；系统可以延后、合并或不执行一次后台机会。自动同步仅为便利功能，不能被表述为定时或实时服务。

## Non-goals

- 不提供 Universe Keyboard 自有账号或托管云服务。
- 不保证第三方文件提供器的后台实时性。
- 不把平台专属皮肤、字体或键位 UI 配置强行应用到其他平台。
- 不自动导入、覆盖或合并任意 YAML 文本；配置导入需单独的 staging、差异预览和恢复安全设计。
- 不将“上传成功”表述为“本地部署成功”。
- 不在没有完成首次手动标准同步、键盘仍活跃、文件夹授权失效或后台任务被系统取消时执行自动标准同步。
