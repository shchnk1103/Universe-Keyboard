## Gates

### Entry Criteria

- Human Product Owner has authorized the release-control structure. **Met**
- Target date and intended channel are explicit. **Met**
- Child tasks have bounded scopes and honest Assignment status. **Met**
- External release actions remain separately authorized. **Met**

### Exit Criteria

This umbrella task may become `Completed` only when:

- the exact release commit/build is frozen;
- every child Assignment is `Closed` or has a Product Lead-approved exclusion/risk record;
- the final release evidence maps to that exact commit and artifact;
- all failed and skipped gates have an owner and Product decision;
- App Store submission state, review result and intended release control are recorded;
- required documentation and `CHANGELOG.md` impacts are complete.

`Completed` does not authorize App Store release. `Closed` requires the Human Product Owner's explicit release decision and complete handoff.

### Stop Conditions

Stop coordination and return to the named authority when:

1. A child task is asked to enter `Ready` or `Active` while a required field remains `UNKNOWN`.
2. A release claim relies on a different commit, beta-only build or historical device snapshot.
3. A child task expands product, privacy, data, deployment or support scope without Product Lead revalidation.
4. A skipped blocker, external upload, submission or release is requested without explicit Human Product Owner approval.
5. Final evidence cannot be tied to the exact archive, dSYM, device/environment and App Store build.

- **Status synchronization rule:** child Assignment and evidence source update first; Dashboard summary updates afterward
- **Revalidation Trigger:** target date/channel change; scope/support-matrix change; release commit change; Apple toolchain/submission requirement change; owner/reviewer reassignment; any accepted skipped gate
